### 06. 데이터 수집 전략 *(이번에 신규 추가)*
`배치.txt`(팀 촬영 계획)를 그대로 슬라이드화:
- **폭파구·불발탄**: 정상 개별 근접 촬영(증강용) → 크기(대/중/소)×배경(잔디/도로) 조합 체계화 → 각도 다양화+그림자 점검 → 배경 세부(유도로 ㄱ/T/一자, 활주로 끝 흰색마킹) 반영 → 촬영 우선순위(폭파구+불발탄 동시 구도, 서 있는 미사일 우선)
- **시설물**: 정상 상태 전체 우선 촬영 → 45도 근접 촬영 → 상태 조합(정상/화재/파손/화재+파손×6종) 전수 커버 → 좋은 각도부터
- 핵심 원칙 배너: "한 zone에는 객체 최대 1개만 배치해 촬영 → 라벨 경계 명확화"

## 3. 이번 세션에서 반영한 슬라이드 노트 피드백

사용자가 슬라이드 1·10 노트에 남긴 항목들을 검토해, **지금 근거 자료로 완성 가능한 것만 우선 반영**했습니다.

### ✅ 반영 완료
- **데이터 수집 슬라이드 필요** → 06번 슬라이드 신규 추가 (`배치.txt` 기반)
- **글씨 크기가 작아 보임** → 여유 있는 슬라이드(02·04·05·06·10·11·12) 본문 텍스트를 0.3~0.5pt씩 확대. 밀도가 이미 높은 03·07·08·09는 레이아웃이 깨질 위험이 있어 유지

### ⏳ 아직 반영 못 한 항목 (근거 데이터가 아직 없어서 보류 — 생기는 대로 계속 채워나갈 것)
- **git 업데이트 지속 반영**: 팀원 push가 계속 이어지는 중이라, 다음에 "지금 커밋 기준으로 갱신해줘"라고 요청하면 슬라이드 03 타임라인·05 모델 정보 등을 그때그때 재동기화
- **테스트/개발 결과, 1차·2차 리허설 결과**: 아직 리허설이 진행되지 않아 실제 수치가 없음. 리허설 끝나면 결과(정확도·소요시간 등) 알려주면 슬라이드 03 또는 새 슬라이드로 반영
- **최종 모델 선정 이유 + 가중치 설명**: 로컬 field_config.py가 팀원 push 반영 전이라 가중치 파일명을 함부로 못 씀(확인 결과 YOLO11n 사용 중이라고 확인해주심 — 슬라이드 05는 이미 YOLO11n으로 정확히 기재되어 있어 그대로 둠). 최종 push 이후 정확한 가중치 파일명·학습 조건이 확정되면 슬라이드 05나 07에 구체적으로 추가 가능
- **자료 이미지로 텍스트 대체**: 팀이 촬영 이미지를 계속 수집 중이라 아직 실제 사진이 없음. 실사진이 준비되면 해당 위치(특히 06 데이터수집, 02 파이프라인)에 이미지 삽입 가능
- **슬라이드 11 촬영 위치/각도/횟수 전략**: "어떤 위치에서 어떤 각도로 몇 번 촬영하고 전송하는지" 구체적 비행 경로가 아직 팀 내부적으로 확정되지 않은 것으로 보여 보류. 결정되면 슬라이드 11(임무 수행 전략)에 구체 동선으로 추가 가능
